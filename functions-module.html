code_searching(input_file, language, operation, comments)

source code 

Search critical operations occurrences in a certain code.

Iterate over the lines of a source code files, search for occurrences of the target operation (division or floating point op), and retrieve positive cases.

Parameters:
  • input_file - source code file to be analyzed.
  • language - ProgrammingLanguage object. Match with the programming language of the source code file.
  • operation - target operation to be searched.
  • comments - flag to either ignore or include source code comments in the analysis process.
Returns:
data to be printed in the results file, composed by the found occurrences in the analyzed code.

Print a file header in the output file.

A file header contains the absolute path of the analyzed file. It is used to separate the results from one file to another.

Parameters:
  • file_name - name of the file being analyzed. @return header to be printed as file separator.

remove_discriminant(target_list, ignored_list, extensions)

source code 

Remove files with invalid extensions.

Remove from the list of files to be analyzed, those with invalid extensions (extensions that are not related to the selected programming language.)

Parameters:
  • target_list - list of files to be analyzed.
  • ignored_list - list with already ignored files (wont be analyzed).
  • extensions - list of allowed extensions.
Returns:
(Tuple) updated ignored_list (files with invalid extensions has been added to the list) and updated target_list (files with invalid extensions have been deleted).

Obtain the absolute paths of the files to be processed.

Obtain the absolute path of each file listed in the file with the targets to be processed.

Parameters:
  • full_list - list with target elements to be processed.
  • ignored_list - list with current ignored elements.
  • recursivity - flag to establish if directories shall be iterated recursively or not.

(Tuple) list with the absolute paths, and updated 'ignored_list' (targets that does not exist has been added).

list_file_retrieval(list_file_list)

Extract targets to be analyzed from the input files.

An input file contains a series of text lines, each one representing either a file or a directory. These lines are extracted and loaded in a list.

Parameters:
